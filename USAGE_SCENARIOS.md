# Product-Level Usage Scenarios (Regulation Manager)

## Purpose
Define real-world usage flows for 교수, 학생, 직원 중심의 웹/MCP 서비스로서
"사용자가 원하는 것을 제대로 판단해서 제공"하는 것을 목표로 한다.

## Channels and Cadence
- Primary: Web UI
- Secondary: MCP (AI agent integration)
- CLI: internal testing only
- Regulation updates: monthly (1x / month)

## Personas and Jobs-to-be-Done
1) 교수
   - 인사/연구년/업적/승진 기준 확인
   - 정확한 조항 근거와 적용 범위 확인
2) 학생
   - 학사(휴학/복학/졸업/장학) 절차 확인
   - 본인 적용 여부 및 필요한 조건 확인
3) 직원
   - 행정/인사/복무/징계 기준 확인
   - 규정 최신성 및 버전 확인

## Core Scenarios (Web)

### S1. 교수: 연구년 신청 자격 확인
- Trigger: "연구년 자격 조건이 뭐지?"
- Flow:
  1) 검색/질문 입력
  2) 시스템이 질의 유형/대상(교수)을 추정
  3) 요약 답변 + 근거 조항(규정명/조항) 표시
  4) "대상 불일치" 신호가 있으면 자연스럽게 안내
  5) 원문 펼쳐보기로 상세 확인
- Success:
  - 1차 답변에서 해당 규정명+조항을 정확히 제시
  - 교수 대상 규정이 우선 노출

### S2. 학생: 휴학 절차 문의
- Trigger: "휴학하려면 어떻게 해야 하나요?"
- Flow:
  1) 질문 입력
  2) 절차 요약 + 단계별 근거 조항 표시
  3) 필요한 조건/기한/서류가 명시된 조항만 인용
  4) 결과 공유(링크/복사)로 행정 문의 대비
- Success:
  - 절차 요약 + 근거 조항이 함께 제공
  - 불필요한 교직원 규정 노출 최소화

### S3. 직원: 인사 규정 검색 (승진/전보)
- Trigger: "승진 요건/전보 기준 확인"
- Flow:
  1) 키워드 검색
  2) 규정명/조항 경로로 결과 정렬
  3) 관련 조항만 빠르게 스캔 가능
  4) 규정 최신성(갱신일/버전) 표시
- Success:
  - 관련 규정 TOP3 내 노출
  - 최신 버전 표시

### S4. "전문" 요청: 특정 규정 전체 보기
- Trigger: "교원인사규정 전문" / "교원인사규정 원문"
- Flow:
  1) 규정명 정합성 확인
  2) 규정 단위 뷰 제공(목차/조항 트리)
  3) 부칙/별표 분리 표시
- Success:
  - 전체 문맥 보기 가능
  - 반복 경로/중복 표시는 제거

### S5. 모호한 질의 (대상/의도 불명확)
- Trigger: "징계 기준 알려줘"
- Flow:
  1) 대상 후보(학생/교원/직원) 제안
  2) 사용자가 선택하면 재검색
  3) 선택된 대상 기반 답변 제공
- Success:
  - 잘못된 대상 규정 노출 최소화
  - 재질문 횟수 감소

## Core Scenarios (MCP)

### M1. 에이전트 통합: 규정 근거 삽입
- Trigger: 업무봇이 "연구년 규정"을 문서에 삽입
- Flow:
  1) MCP search 호출
  2) 규정명/조항/본문 반환
  3) 문서에 근거 삽입 + 원문 링크
- Success:
  - 출처가 명확한 인용
  - 불일치 규정 최소화

### M2. 에이전트 질의 응답
- Trigger: 챗봇이 규정 Q&A 처리
- Flow:
  1) MCP ask 호출
  2) 답변 + 근거 반환
  3) 필요 시 follow-up 질문 생성
- Success:
  - 규정 근거가 포함된 답변
  - 환각 방지(근거 없는 추론 차단)

## Monthly Update Flow
- Trigger: 규정집 업데이트(월 1회)
- Flow:
  1) 변환 → 동기화
  2) 변경 규정 목록 생성
  3) Web/MCP에 "최신 업데이트" 표시
- Success:
  - 최신 규정 반영
  - 변경 이력 추적 가능

## Product Improvements Derived from Scenarios
1) 사용자 대상(교수/학생/직원) 선택 또는 자동 추론 강화
2) 규정 단위 "전문 보기" UX (목차 + 펼치기)
3) 질문/검색 모드 자동 판단 + 근거 기반 요약
4) 업데이트 정보 표시 (마지막 갱신, 버전)
5) 모호 질의 시 후속 질문 UI
6) MCP 응답에 경로 정규화 및 중복 제거

## Success Metrics
- 1차 답변 해결률 (사용자 만족/재질문 없이 종료)
- 근거 조항 클릭률 및 체류 시간
- 대상 불일치 경고 발생률
- 검색 실패율(결과 0 또는 상위 결과 무관)

## Validation Checklist
- 교수/학생/직원 각각 대표 질문 5개씩 통과
- "전문" 요청 시 규정 전체 보기 제공
- 월 1회 업데이트 후 최신 규정 반영 확인
